# Sync-RP
这是一份非常有创意的TRPG（桌面角色扮演游戏）工具。解决了传统跑线上团的核心痛点：传统的QQ群/即时通讯跑团容易导致“刷屏快、逻辑乱、插嘴难、复盘累”。  通过“多列并行输入+KP逐行结算”的逻辑，把“异步交流”和“同步结算”进行有机结合。

一、 核心设计理念

并行输入，串行结算：玩家无需等待他人输入，KP（守秘人/主持人）掌握叙事节奏的主导权。
时空统一性：通过“逐行扫描”机制，确保所有角色的行动发生在同一个时间切片内。
信息分层：区分“场内RP（角色扮演）”、“场外音（OOC）”与“系统结算”。

二、 核心功能模块

1. 多轨并行编辑器 (Multi-track Editor)
分列布局：每个玩家拥有独立的垂直输入列，KP拥有全局视图。
预设缓冲区：玩家可以在KP结算前，提前撰写多行RP内容。
优点：消灭“等待掉线感”，玩家可以专注于打磨自己的动作。
动态修改：在KP“扫描”到该行之前，玩家可以随时编辑已发送的内容，防止逻辑冲突。
2. KP逐行结算系统 (The Scanner)
焦点机制：KP屏幕上有一个“扫描框”，逐行处理各列的动作。
锁定逻辑：被KP选中的行进入“锁定状态”，不可再修改，并同步到所有人的“历史日志”中。
评论式结算：每个玩家动作格后方跟随一个KP反馈格，强制要求KP进行判定或叙述反馈。

计划更新：
战争迷雾与社交机制 (Fog of War)
视线隔离：支持将玩家分组。A组看不见B组的实时输入，直到KP将其合并。
#信息传递：
玩家输入 #传话 [内容]。
该信息会脱离玩家列，出现在对方组的KP提示栏中。

QQ联动：保持QQ群作为“语音/骰子/闲聊”的第二屏幕，减轻网页端的数据压力。
